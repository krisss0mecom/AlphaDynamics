# AlphaDynamics Product V1 Plan

Date: 2026-04-28

This document turns the current research repository into a usable research
product without narrowing or deleting the scientific claims. The product goal is
not "replace molecular dynamics." The product goal is a reproducible,
auditable, per-system torsion-dynamics surrogate workflow.

## Product Position

AlphaDynamics is a command-line research product for training a compact
per-domain phase-flow surrogate from seed MD and auditing the result in torsion
space.

Primary user:

- computational biophysics or ML researcher with mdCATH-style trajectories
- wants a small per-system surrogate, not a zero-shot sequence model
- needs transparent NLL, rollout, and baseline reports

Product promise:

- convert aligned mdCATH HDF5 inputs to phi/psi torsion arrays
- train AlphaDynamics and matched baselines
- evaluate one-step NLL and rollout free-energy fidelity
- generate Markdown audit reports from JSON artifacts
- expose limitations as first-class report fields

## Command Surface

Install editable package:

```bash
pip install -e .
```

Check environment:

```bash
alphadynamics doctor
```

Validate aligned torsion data:

```bash
alphadynamics validate-data \
  --data-dir mdcath_real_data/mdcath_348K \
  --strict
```

Convert raw mdCATH files:

```bash
alphadynamics convert \
  --bench-dir mdcath_raw \
  --out-dir mdcath_real_data/mdcath_348K
```

Run one-step NLL audit:

```bash
alphadynamics train \
  --data-dir mdcath_real_data/mdcath_348K \
  --out-prefix mdcath_aligned20_4000step_cpu \
  --steps 4000 \
  --batch 512 \
  --device auto
```

Run rollout/free-energy audit:

```bash
alphadynamics rollout \
  --data-dir mdcath_real_data/mdcath_alltemps \
  --out-prefix ramachandran_aligned3_4000step_gpu \
  --domains 1lwjA03 1kwgA03 1vq8L01 \
  --steps 4000 \
  --batch 512 \
  --rollout-steps 2500 \
  --kappa-mult 30 \
  --device auto
```

Run stronger residual-baseline audit:

```bash
alphadynamics strong-baseline \
  --data-dir mdcath_real_data/mdcath_348K \
  --out-prefix strong_baseline_audit \
  --steps 4000 \
  --batch 256 \
  --seeds 42 43 44 \
  --phaseflow-tmax 1 4 \
  --device auto
```

Run rollout calibration sweep:

```bash
alphadynamics kappa-sweep \
  --data-dir mdcath_real_data/mdcath_alltemps \
  --out-prefix kappa_sweep_n48 \
  --domains 1lwjA03 1kwgA03 1vq8L01 \
  --kappa-mult 1 5 10 20 30 50 \
  --device auto
```

Build an audit report from shipped result files:

```bash
alphadynamics report \
  --output results/alphadynamics_audit_report.md
```

Every long-running command supports `--dry-run`.

## Research Expansion Without Deleting Claims

The current paper claims stay intact as v1 evidence. The expansion path is an
evidence ladder:

1. Residual MLP audit
   - Run `strong-baseline` on all 40 aligned domains.
   - Use 3 seeds where compute allows.
   - Report wins vs both absolute and residual MLP.

2. Rollout calibration audit
   - Sweep `kappa_mult` in `{1, 5, 10, 20, 30, 50}` on the six rollout domains.
   - Report per-domain optimum and global fixed setting.
   - Keep `kappa x30` as v1 result, add calibration evidence as v2 result.

3. Seed sensitivity
   - Repeat the 40-domain NLL audit with seeds `{42, 43, 44}`.
   - Report confidence intervals for ratio-of-means and win rate.

4. Size expansion
   - Add aligned N=150 and N=200 mdCATH subsets.
   - Keep the current N=48/N=98 tables as the low/mid-size reference.

5. Kinetic and physical observables
   - Add autocorrelation, transition counts, and metastable basin residence
     times where trajectories are long enough.
   - Treat NLL and Ramachandran maps as necessary but not sufficient.

6. External comparison
   - Add an adapter for one public transferable model when the task can be
     matched fairly.
   - Keep the distinction between per-system and transferable models explicit.

## Product Readiness Gates

Ready for public demo:

- `alphadynamics doctor` passes on a clean environment
- `alphadynamics report` regenerates a compact Markdown report from shipped JSON
- `alphadynamics validate-data --strict` passes on aligned local inputs
- one CPU smoke run finishes with `--max-domains 1 --steps 5`

Ready for workshop submission:

- full residual-baseline audit
- kappa calibration table
- seed sensitivity on at least a representative subset
- product report attached to the preprint repository

Ready for external users:

- packaged release with `pip install alphadynamics`
- small downloadable example dataset or synthetic demo
- CI that runs CLI help, report generation, and a tiny smoke train
- model checkpoint save/load path for inference-only use

## Current Product Status

Implemented:

- single command surface: `alphadynamics`
- environment diagnostics: `doctor`
- aligned npz validator: `validate-data`
- one-step NLL wrapper: `train`
- rollout/free-energy wrapper: `rollout`
- rollout calibration wrapper: `kappa-sweep`
- residual-baseline wrapper: `strong-baseline`
- Markdown result report: `report`
- package metadata: `pyproject.toml`

Still research-grade:

- no stable checkpoint/inference API yet
- no public tiny dataset bundled in the repo
- no continuous integration
- long audits still require local mdCATH data and GPU time
